用故事线串联 Wi-Fi 标准的演进，比死记硬背原始数据更有效。

### 1. 远古双雄（`a` 与 `b`）

*   **802.11b**：想到 "**B**asic（基础）" 或 "**B**usy（繁忙）"。它运行在拥挤的 **2.4 GHz** 频段，速度慢（11 Mbps），但因低成本和良好的信号穿透力成为"初代之王"。
*   **802.11a**：想到 "**A**loof（高冷）" 或 "**A**dvanced（先进）"。它使用干净的 **5 GHz** 频段，速度快得多（54 Mbps），采用先进的 **OFDM** 技术。但它是"技术贵族"——价格昂贵、穿透力差，因此普及率低。
*   **关键要点**：`a` 和 `b` 同年发布但走了不同的路。`a` 快但贵；`b` 慢但流行。

> **中文解释**：802.11a 和 802.11b 是最早的两个 Wi-Fi 标准。b 用的是 2.4 GHz，虽然慢但便宜、穿墙好，所以大家都用；a 用的是 5 GHz，虽然快但贵、穿墙差，所以叫好不叫座。记住：**b = 2.4 GHz / 11 Mbps，a = 5 GHz / 54 Mbps / OFDM**。

### 2. 伟大的融合者（`g`）

*   **802.11g**：想到 "**G**ood compromise（好的折中）"。它巧妙地将 `a` 的高速技术（**OFDM**，54 Mbps）与 `b` 的热门频段（**2.4 GHz**）结合起来。关键是它向后兼容 `b`，因此迅速普及。
*   **关键要点**：`g` = `a` 的速度 + `b` 的频段。

> **中文解释**：g 是一个"各取所长"的标准——拿了 a 的速度（54 Mbps + OFDM），用了 b 的频段（2.4 GHz），还能兼容 b 设备。所以 g 一出来就迅速替代了 b。公式记忆：**g = a 的技术 + b 的频段**。

### 3. 革命者（`n` - Wi-Fi 4）

*   **802.11n**：想到 "**N**ew（新）" 或 "**N**imble（灵活）"。它引入了革命性的 **MIMO**（Multiple-Input, Multiple-Output，多输入多输出），把单车道变成多车道高速公路。速度飙升至 600 Mbps。它还很灵活，能同时运行在 **2.4 GHz 和 5 GHz 双频段**。这是第一个被命名为 **Wi-Fi 4** 的标准。
*   **关键要点**：`n` = MIMO + 双频段。

> **中文解释**：n 是真正的革命性升级。MIMO 技术让路由器可以同时用多根天线收发数据，相当于从单车道变成多车道。而且 n 是第一个同时支持 2.4 GHz 和 5 GHz 的标准（双频段）。考试关键词：**n = MIMO = 双频段 = Wi-Fi 4**。

### 4. 加速之王（`ac` - Wi-Fi 5）

*   **802.11ac**：想到 "**AC**celerated（加速）"。它放弃了拥挤的 2.4 GHz 频段，**专注于 5 GHz** 高速公路。它将 MIMO 升级为 **MU-MIMO**（Multi-User MIMO，多用户 MIMO），允许 AP 同时向多个设备发送数据（主要是下行方向）。速度进入 Gbps 时代。
*   **关键要点**：`ac` = 仅 5 GHz + MU-MIMO。

> **中文解释**：ac 只跑 5 GHz，不再支持 2.4 GHz。最大的升级是 MU-MIMO——之前的 MIMO 虽然多天线，但同一时刻只能跟一个设备通信；MU-MIMO 允许 AP 同时跟多个设备通信（但 ac 只支持下行方向的 MU-MIMO）。考试陷阱：如果题目说设备是 ac，AP 只开了 2.4 GHz，那这个设备**连不上**，因为 ac 只支持 5 GHz。**ac = 仅 5 GHz + 下行 MU-MIMO = Wi-Fi 5**。

### 5. 效率大师（`ax` - Wi-Fi 6）

*   **802.11ax**：想到 "e**X**treme efficiency（极致效率）"。它的目标不仅是速度，更是在拥挤环境中保持高效率。它的终极武器是 **OFDMA**（Orthogonal Frequency Division Multiple Access，正交频分多址），将一个信道分割成更小的资源单元，同时服务多个设备。MU-MIMO 也升级为同时支持上行和下行。
*   **关键要点**：`ax` = OFDMA + 高效率。

> **中文解释**：ax 的核心不是"更快"而是"更高效"。OFDMA 把一个大信道切成很多小块（资源单元），每个小块可以分给不同的设备，就像一条大马路被划成很多小车道，每辆小车各走各的，不用等。同时 MU-MIMO 也升级了，上行下行都支持。ax 是为高密度场景设计的（比如体育场、会议室里几百人同时连 Wi-Fi）。**ax = OFDMA + 上下行 MU-MIMO = Wi-Fi 6**。

## 关键词速查联想

*   **OFDM**：起源于 `a` 和 `g` 的调制技术。
*   **MIMO**：立刻想到 `n`。
*   **MU-MIMO**：想到 `ac`（仅下行）和 `ax`（上行 + 下行）。
*   **OFDMA**：`ax`（Wi-Fi 6）的标志性技术。

### 频段分组

*   **仅 2.4 GHz**：`b`、`g`
*   **仅 5 GHz**：`a`、`ac`
*   **双频段（2.4 & 5 GHz）**：`n`、`ax`

> **中文解释**：频段分组是考试高频考点。记忆口诀——**"bg 挤在 2.4，a-ac 独占 5，n-ax 两边跑"**。当题目问兼容性或频段选择时，先确认设备支持哪个频段，再判断能不能连上。

## CCNA 考试重点

CCNA 考试通过场景题考察理解能力，而不仅仅是死记硬背。

### 1. 兼容性问题

*   `g` 设备能连接仅支持 `b` 的网络吗？能，但速度会被限制在 11 Mbps。
*   如果 `b` 设备加入 `g` 网络，由于保护机制（"混合模式惩罚"），整个网络的性能都会下降。
*   `ac` 设备**无法**连接只在 2.4 GHz 广播的 `n` AP，因为 `ac` 是仅 5 GHz 的标准。

> **中文解释**：兼容性的核心逻辑是"向后兼容但会降速"。高版本设备可以连低版本网络，但速度降到低版本的上限。反过来，低版本设备加入高版本网络会拖慢整个网络。特殊情况：ac 只支持 5 GHz，所以如果 AP 只开了 2.4 GHz 的 n，ac 设备根本连不上——这是常见考题陷阱。

### 2. 频段选择

*   **2.4 GHz**：
    *   **优点**：穿透力好（穿墙），覆盖范围广。
    *   **缺点**：非重叠信道少（只有 3 个），干扰大（蓝牙、微波炉等）。
*   **5 GHz**：
    *   **优点**：信道多，干扰少，速度快。
    *   **缺点**：穿透力差，覆盖范围小。
*   **考试场景**：在干扰严重的环境中，应优先选择哪个频段？**答案：5 GHz。**

> **中文解释**：2.4 GHz 的三个非重叠信道是 1、6、11。5 GHz 信道多达 20+ 个。考试常见场景题：用户抱怨 Wi-Fi 慢/不稳定 → 检查是否有干扰 → 如果干扰源多（蓝牙、微波炉），建议切到 5 GHz。

### 3. 核心技术匹配

*   如果题目提到 **Channel Bonding（信道绑定）**（例如将 20 MHz 信道合并为 40 MHz），指的是 **802.11n** 的关键特性。
*   如果题目指定 **下行 MU-MIMO**，指向 **802.11ac**。
*   如果题目提到 **OFDMA**、**高密度部署**或 **IoT 场景**，几乎可以确定是 **802.11ax**。

> **中文解释**：考试做题时看到关键词直接对应标准——Channel Bonding → n，下行 MU-MIMO → ac，OFDMA / 高密度 → ax。这是最高效的解题方式。
