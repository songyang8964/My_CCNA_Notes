Underlay 转发部分

外层以太网头 + 外层 IP 头 + UDP 头。

中间交换机/路由器只看这部分，根据外层目的 IP（VTEP_B 的地址）查路由表做 L3 转发。

它们完全不知道里面包着什么。

VXLAN 头（分界线）

包含 VNI（24 bit），标识这个帧属于哪个逻辑二层网络。

相当于传统 VLAN ID 的升级版，最多约 1600 万个隔离域。

Overlay 的"货物"

用户发出的原始以太网帧，一个字节都没改，完整保留在里面。

目的 Edge Node 解封装后直接把这个原始帧交给目标主机。

转发全流程
Step 1 · Edge Node A
封装
收到用户原始帧 → 查 LISP 得知目标在 Edge B → 在原始帧外面套上 VXLAN头 + UDP头 + 外层IP头 → 发给 Underlay
→
Step 2 · Underlay 网络
L3 转发
中间路由器只看外层 IP 头 → 查路由表 → 逐跳转发 → 不拆包、不看内层 → 送到 Edge Node B
→
Step 3 · Edge Node B
解封装
收到 VXLAN 包 → 剥掉外层所有头部 → 还原出原始以太网帧 → 交给目标主机
核心理解：Underlay 网络中的设备只看到一个普通的 IP/UDP 包在传输，它们不知道里面装着用户的二层帧。Overlay 的二层帧被当作"货物"封装在 L3 包裹里，通过 L3 网络安全送达目的地后再拆开。这就是 "L2 over L3" 的本质。
